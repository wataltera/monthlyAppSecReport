When using Mend (Formerly WhiteSource) dependency analysis tool which identifies vulnerable libraries, what is the difference between the mend command "dependencies: Run dependencies (SCA) scan (alt: mend dep , mend sca)" and the command "ua: Run dependencies (SCA) scan with unified-agent engine?"

mend dep runs after a build, mend ua can run before a build.  ua requires a complex configuration file and takes more computation resources.

Sample mend command for dimpoc SCA scan

\mend\mend.exe dep --mend-url https://saas.mend.io/ -s 'sandbox//dimpoc' -u <user key> -k <api key> --source <path> defaults to .  

Needs --build "build command" if it has to do its own build.  Leave this off if it's been built.

mend sast for source vulnerabilities.

mend sca --source ./my-app -u <your_user_key> -k <your_api_key> --project <project_name>

Measure-Command { \mend\mend.exe sca -s 'sandbox//dimpoc' }
Measure-Command { \mend\mend.exe sast -s 'sandbox//dimpoc' }
